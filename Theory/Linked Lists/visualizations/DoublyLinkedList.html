Doubly Linked List

A bidirectional linear data structure with forward and backward traversal capabilities

Bidirectional Navigation: Each node contains data and two pointers - one pointing to the next node and another to the previous node, enabling traversal in both directions.

Key Characteristics

  • Bidirectional: Can traverse both forward and backward
  • Dynamic Size: Can grow or shrink during runtime
  • Dual Pointers: Each node has prev and next pointers
  • Efficient Deletion: Can delete a node in O(1) if reference is known
  • Memory Overhead: Requires extra memory for previous pointers

Common Operations

Insert at Front

Add a new node at the beginning of the list

Time: O(1) | Space: O(1)
Insert at End

Add a new node at the end of the list

Time: O(1) with tail | O(n) without tail
Insert at Position

Add a new node at a specific index

Time: O(n) | Space: O(1)
Delete at Front

Remove the first node from the list

Time: O(1) | Space: O(1)
Delete at End

Remove the last node from the list

Time: O(1) with tail | O(n) without tail
Delete by Reference

Remove a node when you have its reference

Time: O(1) | Space: O(1)
Forward Traversal

Print list from head to tail

Time: O(n) | Space: O(1)
Backward Traversal

Print list from tail to head

Time: O(n) | Space: O(1)

Node Structure

class DoublyNode { int data; // Stores the actual value DoublyNode prev; // Points to the previous node DoublyNode next; // Points to the next node // Constructor DoublyNode(int data) { this.data = data; this.prev = null; this.next = null; } }

Pros & Cons

✓ Advantages

  • • Bidirectional traversal
  • • Efficient deletion with reference
  • • Better for undo operations
  • • Can traverse backward easily
  • • Insertion/deletion at both ends is O(1)

✗ Disadvantages

  • • Extra memory for prev pointers
  • • More complex pointer manipulation
  • • Not cache-friendly
  • • Higher space complexity
  • • More complex implementation
