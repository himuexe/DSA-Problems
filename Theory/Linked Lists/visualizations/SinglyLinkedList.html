Singly Linked List

A linear data structure where elements are stored in nodes, connected through pointers

Head points to the first node. Each node contains data and a pointer to the next node.

Key Characteristics

  • Unidirectional: Can only traverse forward from head to tail
  • Dynamic Size: Can grow or shrink during runtime
  • Sequential Access: Must traverse from head to reach any element
  • Memory Efficient: No wasted memory, allocates as needed
  • Non-Contiguous: Nodes can be stored anywhere in memory

Common Operations

Insert at Front

Add a new node at the beginning of the list

Time: O(1) | Space: O(1)
Insert at End

Add a new node at the end of the list

Time: O(n) | Space: O(1)
Insert at Position

Add a new node at a specific index

Time: O(n) | Space: O(1)
Delete at Front

Remove the first node from the list

Time: O(1) | Space: O(1)
Delete at End

Remove the last node from the list

Time: O(n) | Space: O(1)
Search

Find a node with specific data value

Time: O(n) | Space: O(1)

Pros & Cons

✓ Advantages

  • • Dynamic size allocation
  • • Memory efficient
  • • Easy insertion/deletion at front
  • • No memory waste

✗ Disadvantages

  • • No random access
  • • Extra memory for pointers
  • • Sequential traversal only
  • • Not cache-friendly
